# GitHub Pages 設定手順（画像付き説明）

## ステップ4の詳細手順

### 1. Settingsタブをクリック
リポジトリのトップページで、上部メニューの「Settings」をクリックします。

### 2. Pagesメニューを開く
左サイドバーの「Pages」をクリックします。

### 3. Sourceを設定
以下のように設定します：
- **Branch**: `main` を選択
- **Folder**: `/ (root)` を選択
- 「**Save**」ボタンをクリック

### 4. 公開完了を待つ（1-2分）
ページ上部に緑色のボックスが表示され、以下のメッセージが出ます：

```
✓ Your site is live at https://<your-username>.github.io/talent-market-monitor/
```

### 5. URLをクリック
表示されたURLをクリックすると、サイトが公開されています！

---

## 🎯 完成したURL

```
https://<your-username>.github.io/talent-market-monitor/
```

（`<your-username>` はあなたのGitHubユーザー名）

---

## 📱 例

ユーザー名が `tanaka123` の場合：
```
https://tanaka123.github.io/talent-market-monitor/
```

このURLをクリックするだけで、誰でもサイトを見られます！

---

## 🔄 サイトを更新する方法

ファイルを修正した後：
1. GitHubリポジトリページで「Add file」→「Upload files」
2. 更新したファイルをドラッグ&ドロップ
3. 「Replace existing files」にチェック
4. 「Commit changes」をクリック

数分後、変更が自動的に公開されます。

---

## 💡 カスタムドメインを使う場合

Settings > Pages > Custom domain で独自ドメインを設定できます：
- 例: `talent-monitor.com`
- DNSレコードの設定が必要です（GitHubが手順を表示します）
